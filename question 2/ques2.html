Q2. This exercise helps understand the difference between inline, block, and inline-block display types. On an HTML page add 3 span elements each with the words HTML, CSS, and JavaScript. Select the 3 elements using the span selector in an embedded style tag on the page and set the width to 40%.

1) How do the elements appear? Does the width have any effect? Why / Why not?

=> The elements appear in inline form. No, the width doesn't have any effect.

2) Set display to inline. Does it cause any change? Why / Why not?

=> It does not cause any changes because span is an inline container. This is because we are trying to change inline element to inline itelf.

3) Change display to block. What change does it cause?

=> It is now a block element that makes span according to row. In block element we can also specify the width and height according to our need.

4) Set display to inline-block. What change does it cause?

=> It is an inline-block element we can specify the width and height according to our need which is not possible in inline element.

5) In conclusion, summarize the differences you noticed in the 3 behavior of the 3 display types. Search for other differences and jot them down mentally / physically!

* Inline element => In inline element all the elements are placed horizontally side by side. An inline element does not start on a new line. In this type we cannot change the height and width of the element.

* Block element => A block element occupies the entire horizontal space of its parent element, and vertical space equal to the height of its contents. In this type we can change the height and width of the element

* Inline-block element => In inline-block all the elements are placed horizontally side by side. We can set height and width values
